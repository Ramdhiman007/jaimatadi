// 2. By Name or ID

// 3. By WebElement

driver.switchTo().frame(iframeElement);

// VERY IMPORTANT: Switch back to main page after work is done!
driver.switchTo().defaultContent();

26. What is Selenium Grid?
English:
Selenium Grid is a tool used to run Selenium test cases on multiple machines and multiple browsers in parallel.

It allows testers to execute the same test scripts on different environments (browser, OS, versions) at the same time.

This helps in:
  • Reducing test execution time
  • Cross-browser testing
  • Running tests on multiple systems
Selenium Grid follows a Hub and Node architecture.
  • Hub → Central server that controls test execution
  • Node → Machines connected to the hub where tests run
How Selenium Grid Works:
  1. Test script is sent to the Hub
  2. Hub checks which Node is available
  3. Hub forwards the test to that Node
  4. Test executes on the browser of that machine
  5. Result is sent back to the Hub
Example:
You can run the same test on:
  • Chrome – Windows
  • Firefox – Windows
  • Safari – Mac
  • Edge – Windows
At the same time. This is called Parallel Testing.

Advantages of Selenium Grid:
  • Parallel test execution
  • Saves execution time
  • Cross-browser testing
  • Cross-platform testing

Hinglish:
Advantages (Hinglish):
  • Ek saath multiple test cases run kar sakte hain
  • Execution time kam ho jata hai
  • Different browsers me testing possible hai
  • Different OS (Windows, Mac, Linux) par testing kar sakte hain

Simple Java Example: 
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.remote.RemoteWebDriver;
import org.openqa.selenium.chrome.ChromeOptions;

import java.net.URL;

public class GridExample {

        URL url = new URL("http://localhost:4444/wd/hub");

        System.out.println(driver.getTitle());

        driver.quit();
    }
}

    • Choose the project where you want to report the defect.

  4. Select Issue Type
    • Choose Bug as the issue type.
  5. Fill Required Fields
    Field Description
    Summary Short description of the bug
    Description Detailed explanation of the issue
    Steps to Reproduce Steps to recreate the bug
    Expected Result What should happen
    Actual Result What actually happened
    Priority Low / Medium / High / Critical
    Environment Browser, OS, build version
    Attachment Screenshot or video proof
    Assignee Developer responsible to fix
  6. Attach Screenshot / Evidence
    • Screenshot, Logs, Screen recording
  7. Click Create
    • Once all details are filled, click Create. The bug will be assigned a unique bug ID (Example: BUG-1025).

29. Explain About Agile Process
Agile is a software development process where the project is developed in small iterations called sprints.
Each sprint usually lasts 2–4 weeks, and in every sprint the team delivers a working piece of software.

Agile focuses on:
  • Continuous improvement
  • Customer collaboration
  • Faster delivery
  • Flexibility to handle changes

Key Phases of Agile Process:
  1. Product Backlog
    • English: A list of all features, tasks, and requirements for the project.
    • Hinglish: Product backlog me project ki saari requirements aur features ki list hoti hai.
      Example: Login feature, User registration, Dashboard page
  2. Sprint Planning
    • English: In this meeting the team decides which backlog items will be developed in the next sprint.
    • Hinglish: Sprint planning meeting me decide hota hai ki next sprint me kaun-kaun se tasks develop honge.
  3. Sprint Execution
    • English: Developers develop features and testers test them during the sprint.
    • Hinglish: Is phase me developers feature develop karte hain aur testers testing karte hain.
  4. Daily Stand-up Meeting
    • English: A short meeting (15 minutes) where team members discuss:
      • What did I do yesterday?
      • What will I do today?
      • Any blockers?
    • Hinglish: Daily stand-up meeting me team discuss karti hai:
      • Kal kya kaam kiya
      • Aaj kya kaam karenge
      • Koi issue/blocker hai ya nahi
  5. Sprint Review
    • English: The team demonstrates the completed work to the Product Owner or client.
    • Hinglish: Sprint review me team client ya product owner ko completed features dikhati hai.
  6. Sprint Retrospective
    • English: Team discusses what went well and what needs improvement.
    • Hinglish: Team discuss karti hai:
      • Kya achha hua
      • Kya improve karna chahiye next sprint me

Agile Roles:
Role Responsibility
Product Owner Manages product backlog
Scrum Master Ensures Agile process is followed
Development Team Develops and tests the software

Advantages of Agile:
English:
  • Faster delivery
  • Continuous feedback
  • Flexible to requirement changes
  • Better collaboration
Hinglish:
  • Fast delivery hoti hai
  • Client feedback jaldi milta hai
  • Requirements change easily handle ho jati hain
  • Team collaboration better hota hai
